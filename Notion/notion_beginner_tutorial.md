# Notion 初學者完整教學：從「紀分找」建立你的數位大腦

**目標受眾：** Notion 初學者 (從零開始)
**核心概念：** 紀 (紀錄) - 分 (分類) - 找 (找回)

---

## 導論：為什麼要學 Notion？

Notion 是一款集筆記、任務管理、知識庫和資料庫於一身的多功能工具。它最大的優勢在於**極高的自由度**和**強大的資料庫功能**，能讓你將分散在各處的資訊（筆記本、行事曆、待辦清單、Excel 表格）整合到一個統一的系統中。

對於初學者來說，Notion 就像一張空白畫布，可能會讓人感到不知所措。但只要掌握核心思維，你就能打造出一個完全屬於自己的數位工作空間。

我們將透過「**紀分找**」這個核心概念，帶你一步步建立你的 Notion 系統。

| 概念 | 核心意義 | Notion 實作功能 |
| :--- | :--- | :--- |
| **紀 (紀錄)** | 先行動，再修正。將所有想法、資訊、資料無差別地寫下來。 | 頁面 (Page)、基本區塊 (Block) |
| **分 (分類)** | 釐清需求，一次只做二元分類。將紀錄的內容結構化。 | 資料庫 (Database)、屬性 (Property) |
| **找 (找回)** | 建立關聯，讓資料開始對話。透過篩選、公式、關聯快速提取資訊。 | 關聯 (Relation)、匯總 (Rollup)、公式 (Formula) |

---

## 第一課：紀 (紀錄) - 從空白頁面開始

「紀」是所有行動的起點。重點不在於你記下了什麼，而在於你**開始記錄**這個行動本身。

### 1. 建立你的第一個頁面

在 Notion 中，所有內容都以「**頁面 (Page)**」的形式存在。

*   **操作：** 點擊左側邊欄的 `+ New Page` 或在任何頁面中輸入 `/page`。
*   **頁面元素：**
    *   **標題：** 頁面的名稱。
    *   **圖示 (Icon) 和封面 (Cover)：** 讓頁面更具辨識度。
    *   **內容區：** 頁面主體，由各種「區塊 (Block)」組成。

### 2. 認識基本區塊 (Block)

Notion 的內容是由一個個區塊堆疊而成。在頁面中輸入 `/` 即可呼叫出所有區塊選項。

| 常用區塊 | 說明 | 快速鍵 |
| :--- | :--- | :--- |
| **Text** | 一般文字內容。 | 直接輸入 |
| **Heading 1/2/3** | 標題，用於區分層級。 | `/h1`, `/h2`, `/h3` |
| **To-do list** | 待辦清單。 | `/todo` 或 `[]` |
| **Toggle list** | 可展開/收合的清單，用於隱藏細節。 | `/toggle` 或 `>` |
| **Quote** | 引用文字。 | `/quote` |
| **Callout** | 醒目提示框，用於強調重要資訊。 | `/callout` |

**💡 紀的思維：** 看到任何靈感、想法、待辦事項，**立刻**用最快的速度記錄下來，不用擔心格式。

## 第二課：分 (分類) - 建立你的資料庫

當你的紀錄累積到一定數量，就需要「分」來結構化這些資訊。「分」的核心是將內容放入「**資料庫 (Database)**」這個「書櫃」中。

### 1. 資料庫與頁面的區別

*   **頁面 (Page)：** 獨立的筆記本，內容是自由的區塊。
*   **資料庫 (Database)：** 結構化的表格，每一列（Entry）都是一個獨立的頁面，且擁有固定的「屬性 (Property)」。

### 2. 建立資料庫

*   **操作：** 在頁面中輸入 `/database`，選擇 `Table - Full page` (全頁資料庫) 或 `Table - Inline` (內嵌資料庫)。
*   **資料庫屬性 (Property)：** 每一欄都是一個屬性，用來描述該筆紀錄的特性。

| 常用屬性類型 | 說明 | 應用範例 |
| :--- | :--- | :--- |
| **Text** | 儲存一般文字。 | 筆記內容、摘要 |
| **Number** | 儲存數字，可設定格式 (如貨幣、百分比)。 | 金額、數量、分數 |
| **Select / Multi-select** | 預設選項的單選/多選標籤。 | 專案狀態、文章分類、心情標籤 |
| **Date** | 儲存日期和時間。 | 建立日期、截止日期、活動時間 |
| **Checkbox** | 是/否的勾選框。 | 是否完成、是否已讀 |
| **URL / Email / Phone** | 儲存特定格式的連結。 | 參考資料連結 |

**💡 分的思維：** 像記帳一樣，先區分出「**數字相關**」（金額、日期）和「**非數字相關**」（類別、地點），並為它們建立合適的屬性。

### 3. 資料庫視圖 (View) - 讓資料庫活起來

資料庫的強大之處在於，你可以用不同的「視圖」來呈現同一組資料。

| 視圖類型 | 說明 | 應用場景 |
| :--- | :--- | :--- |
| **Table (表格)** | 傳統的試算表格式，適合大量資料輸入和總覽。 | 記帳明細、資產清單 |
| **Board (看板)** | 類似 Trello，以卡片形式按屬性 (如狀態) 分組。 | 專案進度管理 (To Do, Doing, Done) |
| **Calendar (日曆)** | 以日期屬性為基礎，將資料顯示在日曆上。 | 行程規劃、內容發布排程 |
| **Gallery (畫廊)** | 以卡片形式顯示，適合視覺化內容。 | 食譜庫、電影清單、靈感收集 |
| **List (清單)** | 簡潔的清單模式，只顯示最關鍵的屬性。 | 閱讀清單、快速筆記列表 |

## 第三課：找 (找回) - 讓資料庫開始對話

「找」是 Notion 系統的精華，它能讓你從海量資訊中，快速提取出你需要的「情境與觸動」。

### 1. 屬性關聯 (Relation) - 建立資料庫之間的連結

關聯屬性是將兩個獨立的資料庫連結起來的橋樑，實現「資料不會說話，但關聯會讓它們開始對話」的效果。

**應用場景：** 建立一個「專案」資料庫和一個「任務」資料庫。

1.  在「任務」資料庫中新增一個 `Relation` 屬性。
2.  選擇連結到「專案」資料庫。
3.  現在，每條任務紀錄都可以連結到它所屬的專案。

### 2. 匯總 (Rollup) - 跨資料庫的統計與計算

匯總屬性必須依賴於「關聯」屬性。它的作用是從被關聯的資料庫中，提取特定屬性的值，並進行統計計算。

**應用場景：** 統計每個「專案」下有多少「任務」已完成。

1.  在「專案」資料庫中新增一個 `Rollup` 屬性。
2.  選擇關聯的資料庫 (`Relation`)：選擇連結到「任務」資料庫的屬性。
3.  選擇屬性 (`Property`)：選擇「任務」資料庫中的 `Checkbox` (是否完成) 屬性。
4.  選擇計算方式 (`Calculate`)：選擇 `Count checked` (計算已勾選的數量)。

### 3. 公式 (Formula) - 讓資料動起來

公式屬性是 Notion 最強大的功能之一，它能讓你根據其他屬性的值進行計算、判斷和文字處理。

**💡 找的思維：** 利用公式，將你「找」回來的資料進行二次加工，變成更具洞察力的資訊。

#### 範例一：自動判斷任務狀態 (IF 判斷式)

假設你有一個 `Date` 屬性 (截止日期) 和一個 `Checkbox` 屬性 (是否完成)。

**目標：** 建立一個 `Status` 公式屬性，如果已完成則顯示 "✅ 已完成"，如果未完成且截止日期已過則顯示 "🚨 已逾期"，否則顯示 "⏳ 進行中"。

```notion
if(prop("是否完成"), "✅ 已完成", if(prop("截止日期") < now(), "🚨 已逾期", "⏳ 進行中"))
```

| 屬性名稱 | 屬性類型 | 範例值 |
| :--- | :--- | :--- |
| `是否完成` | Checkbox | `true` (已勾選) |
| `截止日期` | Date | `2025/11/01` |
| `Status` | Formula | `✅ 已完成` |

#### 範例二：計算距離截止日期的天數 (日期計算)

假設你有一個 `Date` 屬性 (截止日期)。

**目標：** 建立一個 `Days Left` 公式屬性，計算距離截止日期還有多少天。

```notion
dateBetween(prop("截止日期"), now(), "days")
```

| 屬性名稱 | 屬性類型 | 範例值 |
| :--- | :--- | :--- |
| `截止日期` | Date | `2025/11/15` |
| `Days Left` | Formula | `10` (天) |

#### 範例三：進度條顯示 (文字處理與數字格式化)

假設你有一個 `Number` 屬性 (進度百分比，0-100)。

**目標：** 建立一個 `Progress Bar` 公式屬性，用視覺化的方式顯示進度。

```notion
slice("■■■■■■■■■■", 0, floor(prop("進度百分比") / 10)) + slice("□□□□□□□□□□", 0, 10 - floor(prop("進度百分比") / 10)) + " " + format(prop("進度百分比")) + "%"
```

| 屬性名稱 | 屬性類型 | 範例值 |
| :--- | :--- | :--- |
| `進度百分比` | Number | `70` |
| `Progress Bar` | Formula | `■■■■■■■□□□ 70%` |

## 第四課：進階應用 - 找回被遺忘的「情境與觸動」

您提到的「找」的延伸，是希望透過更複雜的屬性來建立「**關鍵字**」，以便未來能找回重要的資料。這正是 Notion 關聯資料庫和公式的進階應用。

### 1. 建立「情境/觸動」資料庫

為了讓資料庫之間能夠「對話」，我們需要一個專門的資料庫來儲存這些「關鍵字」或「情境」。

*   **資料庫名稱：** `情境與觸動 (Context & Trigger)`
*   **主要屬性：**
    *   `Name` (標題)：情境名稱 (例如：工作壓力大、週末旅行、讀書心得、靈光一閃)
    *   `Type` (Select)：類型 (情緒、活動、主題)
    *   `Keyword` (Text)：核心關鍵字

### 2. 連結你的主要資料庫

將你的主要資料庫（例如：`日記`、`任務`、`筆記`）透過 `Relation` 屬性連結到 `情境與觸動` 資料庫。

*   **在 `日記` 資料庫中新增屬性：** `Relation` -> 連結到 `情境與觸動`。

### 3. 利用公式屬性提取「找回」的關鍵字

現在，我們可以在 `日記` 資料庫中，利用 `Rollup` 和 `Formula` 屬性，將連結到的「情境」關鍵字自動提取出來，作為一個可供搜尋的欄位。

#### 步驟一：匯總 (Rollup) 關鍵字

在 `日記` 資料庫中新增一個 `Rollup` 屬性：

*   **關聯 (Relation)：** 選擇連結到 `情境與觸動` 的屬性。
*   **屬性 (Property)：** 選擇 `情境與觸動` 資料庫中的 `Keyword` 屬性。
*   **計算方式 (Calculate)：** 選擇 `Show original` (顯示原始值) 或 `Join and separate by ","` (以逗號連接)。

#### 步驟二：公式 (Formula) 強化搜尋

如果你想讓這個關鍵字欄位更醒目，可以利用公式將其格式化。

**目標：** 將匯總回來的關鍵字加上標籤符號 `#`，並以空格分隔。

```notion
// 假設 Rollup 屬性名稱為 "匯總關鍵字"
if(prop("匯總關鍵字"), "#" + replaceAll(prop("匯總關鍵字"), ",", " #"), "")
```

**最終效果：**

當你在日記中連結了「工作壓力大」和「靈光一閃」兩個情境，這個公式欄位就會自動顯示：`#工作壓力大 #靈光一閃`。

**「找」的實現：**

當你未來想找回「工作壓力大」時的紀錄，你只需要在 `日記` 資料庫的搜尋欄中輸入 `#工作壓力大`，就能快速篩選出所有相關的日記，從而找回當時的「情境與觸動」。

## 總結與下一步

這份教學涵蓋了 Notion 從基礎頁面到進階資料庫關聯和基本公式的應用。

| 紀分找階段 | 核心操作 | 關鍵功能 |
| :--- | :--- | :--- |
| **紀 (紀錄)** | 隨手寫下想法 | 頁面、基本區塊 |

| **找 (找回)** | 建立連結與自動化 | 關聯、匯總、公式 |

3.  **持續優化：** Notion 系統是活的，隨著你的需求變化，隨時調整你的資料庫結構和屬性。
