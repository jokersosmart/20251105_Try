# RPD \- 社群平台爬蟲工具

* **專案名稱：** 社群留言資料匯出工具  
* **專案代號：** (內部使用)  
* **文件版本：** v1.0  
* **日期：** 2025年10月8日  
* **專案關係人：** 馮導 (錨點影音 CEO)

---

#### **1\. 專案概述 (Project Overview)**

本專案旨在開發一個桌面應用程式，讓使用者（特別是不具備程式背景的操作人員）能夠輕易地從指定的社群平台 (Instagram & Facebook) 爬取資料，並將結果匯出為單一的 Excel 檔案。此工具的目的是簡化資料收集流程，以利後續進行客戶留言分類、回應追蹤等商業分析。

#### **2\. 核心需求 (Core Requirements)**

**2.1. 功能性需求 (Functional Requirements)**

| ID | 需求名稱 | 需求描述 | 優先級 |
| :---- | :---- | :---- | :---- |
| **FR-01** | **資料來源平台** | 系統必須能夠爬取以下兩個平台的公開資料： 1\. Instagram (IG) 2\. Facebook (FB) | **高** |
| **FR-02** | **使用者友善介面** | 系統需提供一個簡單直觀的圖形化使用者介面 (GUI)，讓使用者可以透過點擊按鈕、輸入目標網址等方式操作，無需撰寫任何程式碼。 | **高** |
| **FR-03** | **資料匯出格式** | 系統完成資料爬取後，需將所有結果整理並匯出成一個 `.xlsx` 格式的 Excel 檔案。 | **高** |
| **FR-04** | **獨立執行檔** | 最終交付的程式需要被打包成一個獨立的執行檔（例如 Windows 的 `.exe` 或 macOS 的 `.app`），使用者下載後即可直接運行，無需安裝複雜的環境（如 Python、相關套件等）。 | **高** |

**2.2. 非功能性需求 (Non-Functional Requirements)**

| ID | 需求名稱 | 需求描述 |
| :---- | :---- | :---- |
| **NFR-01** | **易用性** | 產品設計應以「讓不會寫程式的人也能用」為最高原則，操作流程應盡可能簡化。 |
| **NFR-02** | **可靠性** | 程式在正常網路環境下應能穩定運行，並能處理基本的錯誤情況（例如目標頁面不存在）。 |

#### **3\. 未來可能擴充功能 (Potential Future Enhancements)**

根據馮導提及，專案未來可能有第二階段的擴充需求，初期開發可將其納入架構考量：

* **留言管理系統：** 開發更進階的功能，用於對爬取下來的留言進行分類、標記、以及追蹤回應狀態。

#### **4\. 待釐清問題與假設 (Assumptions and Questions for Clarification)**

為了提供更精準的時程與預算評估，需要與馮導進一步確認以下細節：

**4.1. 關於資料範圍：**

* **具體目標：** 需要爬取的是粉絲專頁、公開社團、還是特定個人帳號的貼文？  
* **資料欄位：** 需要爬取哪些具體資料？例如：貼文內容、發文時間、按讚數、留言數、留言者ID、留言內容、留言時間等。  
* **資料深度：** 是否需要爬取留言的回覆串？需要爬取多少數量的貼文或留言？

**4.2. 關於技術細節：**

* **登入需求：** 爬取目標是否需要登入帳號才能查看？（這會增加開發的複雜度）  
* **目標平台：** 使用者主要在哪個作業系統上運行此程式？ (Windows 或 macOS)

**4.3. 關於使用者操作：**

* **輸入方式：** 使用者將如何指定爬取目標？是貼上網址，還是輸入帳號名稱？  
* **執行頻率：** 這個工具是會每天頻繁使用，還是偶爾執行一次？

#### **5\. 專案交付物 (Deliverables)**

1. **桌面應用程式執行檔：** 一個可獨立運行的爬蟲工具程式。  
2. **簡易操作說明文件：** 一份指導使用者如何安裝與操作此工具的說明書。

### **1\. 專案概述 (Project Overview)**

![社群留言爬蟲系統架構流程圖][image1]  
本專案旨在開發一個線上的應用程式，讓使用者（特別是不具備程式背景的操作人員）能夠輕易地從指定的社群平台（Instagram & Facebook）公開貼文爬取留言資料 。此工具為一通用爬蟲，使用者可透過貼上目標網址來指定任何公開的粉絲專頁或帳號進行資料擷取 。系統支援將爬取結果進行線上編輯，並將最終資料匯出為單一的 Excel 檔案 。此工具的目的是簡化資料收集流程，以利後續進行客戶留言分類、回應追蹤等商業分析 。

### **2\. 核心需求 (Core Requirements)**

#### **2.1. 功能性需求 (Functional Requirements)**

| ID | 需求名稱 | 需求描述 | 優先級 |
| :---- | :---- | :---- | :---- |
| FR-01 | 資料來源平台 | 系統必須能夠爬取 Instagram (IG) 與 Facebook (FB) 兩個平台的公開資料 。 | 高 |
| FR-02 | 使用者友善介面 | 系統需提供簡單直觀的圖形化使用者介面(GUI)，讓使用者可透過輸入目標貼文網址與點擊按鈕等方式操作，無需撰寫任何程式碼 。 | 高 |
| FR-03 | 編輯功能 | 系統完成資料爬取後，需提供介面讓使用者能手動填寫或修改「回覆窗口」、「回覆內容」、「客戶確認/修改處」等欄位，並可刪除不需要的留言資料列。 | 高 |
| FR-04 | 資料匯出格式 | 系統需將所有整理後的結果匯出成一個 .xlsx 格式的 Excel 檔案 。 | 高 |
| FR-05 | 線上應用程式 | 最終交付的程式需要是一個 Webapp（線上應用程式），讓使用者可透過瀏覽器直接操作 。 | 高 |

| ID | 需求名稱 | 需求描述 |
| :---- | :---- | :---- |

### **3\. 未來可能擴充功能 (Potential Future Enhancements)**

根據馮導提及，專案未來可能有第二階段的擴充需求，初期開發可將其納入架構考量 ：

* **留言管理系統**: 開發更進階的功能，用於對爬取下來的留言進行分類、標記、以及追蹤回應狀態 。  
* **責任分類機制**: 直接根據問題去分類責任人 。  
* **自動回應機制**: 針對常見的 FAQ 問題，自動在 Excel 中生成建議的回覆答案 。  
* **架構擴充性**: 開發架構應保留未來新增功能的彈性與空間 。

### **4\. 匯出 Excel 欄位定義 (Data Format Definition)**

#### **4.1. 資料範圍與深度 (Data Scope & Depth)**

* **爬取目標**: 使用者透過貼上網址，指定 Facebook 或 Instagram 上任一**公開貼文**的留言 。  
* **爬取數量**: 單次操作僅針對一個指定網址的貼文進行爬取 。  
* **資料深度**: 爬取目標貼文下的**第一層所有留言** 。為簡化初期開發複雜度，暫不爬取留言的回覆串（第二層之後的留言）。  
* **登入需求**: 以爬取無需登入即可查看的公開資料為主，以降低操作複雜度 。

#### **4.2. 欄位定義表**

最終匯出的 Excel 檔案將包含以下欄位，並已註明各欄位的資料來源：

| 欄位名稱 (中文) | 資料來源 | 備註 |  |
| :---- | :---- | :---- | :---- |
| 發文時間 | 自動抓取 | 原始貼文的發布時間 。 |  |
| 貼文連結 | 自動抓取 | 使用者輸入的目標網址。 |  |
| 貼文內容 | 自動抓取 | 原始貼文的內文 。 |  |
| 按讚數 | 自動抓取 | 原始貼文的按讚總數 。 |  |
| 留言總數 | 自動抓取 | 原始貼文的留言總數 。 |  |
| 留言時間 | 自動抓取 | 該則留言的發布時間 。 |  |
| 留言者 ID | 自動抓取 | 發表留言的使用者帳號 ID 。 |  |
| 留言內容 | 自動抓取 | 該則留言的文字內容 。 |  |
| 回覆窗口 | **使用者手動填寫** | 此欄位為線上編輯功能的重點，用於指派處理人員。 |  |
| 回覆內容 | **使用者手動填寫** | 記錄預計回覆客戶的內容。 |  |
| 客戶確認/修改處 | **使用者手動填寫** | 記錄與客戶溝通或需要特別注意的事項。 |  |
| 生成的回覆 \[未來\] |  | **未來功能** | 預留欄位，未來由自動回應機制生成。 |
